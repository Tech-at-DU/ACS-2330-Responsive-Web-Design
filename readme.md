# ACS 2330: Responsive Web Design

## Course Description

Most web traffic now comes from mobile devices. This course teaches students to build websites that work well across all screen sizes — from phones to desktops. Topics include mobile-first development, CSS layout, responsive patterns, accessibility, Tailwind CSS, and UX testing. Students will apply these skills to real projects using React.

## Learning Objectives

Students by the end of the course will be able to:

1. Plan a responsive website using wireframes and implementation checklists
2. Build responsive layouts using CSS media queries, Flexbox, and Grid
3. Implement a responsive navigation component including a hamburger menu
4. Apply accessibility standards including semantic HTML, ARIA attributes, and color contrast
5. Use Tailwind CSS to build responsive interfaces
6. Conduct and document a UX test and apply findings to a project

### What each assignment covers

**A1 — SFPOPOS Wireframes** (classes 1–2)
Planning document for making the SFPOPOS site responsive. Includes a mobile audit identifying real problems, a content outline, annotated wireframes for every page at both mobile and desktop sizes, and a CSS implementation checklist mapping each annotation to a CSS approach.

**A2 — Grid Challenge** (class 5)
An external coding challenge focused on responsive CSS Grid layouts. Submitted to Gradescope separately from other work.

**A3 — SFPOPOS Responsive Site** (classes 3–9)
The SFPOPOS site progressively improved over six classes. Final submission includes: mobile-first media queries, responsive Flexbox and Grid layouts, a working hamburger menu with ARIA, semantic HTML and accessibility improvements passing Lighthouse ≥ 90, and all layout rebuilt with Tailwind CSS.

**A4 — UX Test Report + Project 2 Wireframes** (class 10)
Documents a UX test conducted on Project 2 wireframes. Includes tasks given, observations from testers, a summary of what worked and what confused users, and updated wireframes reflecting changes made in response.

**A5 — Project 2** (classes 10–14)
An original responsive website of the student's choosing, built with Tailwind CSS. Must be mobile-first, pass accessibility standards, and incorporate UX test findings. Scored on quality of user experience across mobile and desktop. See [project-2.md](./project-2.md) for rubric.
